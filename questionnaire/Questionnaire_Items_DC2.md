# DC2 Post-Task Questionnaire Items

The questionnaire was administered in Dutch immediately after task completion. Items below are translated to English. Item numbering matches the order as administered (22 items total in the written questionnaire).

**Pre-task confidence** was collected verbally by the researcher before the task began (not part of the written questionnaire):
> "How confident are you right now that you can set up an automation using this type of tool? Give a number from 1 to 7 — 1 is not at all confident, 7 is very confident."

---

## Post-Task Questionnaire (Written)

### Item 1 — Post-Task Confidence
*Researcher-designed. Rated 1 (not at all) to 7 (very confident).*

1. How confident are you that you set up the automation correctly?

---

### System Usability Scale (SUS) — Items 2–11
*Source: Brooke, J. (1996). SUS: A "quick and dirty" usability scale. In P. W. Jordan, B. Thomas, B. A. Weerdmeester, & A. L. McClelland (Eds.), Usability evaluation in industry (pp. 189–194). Taylor & Francis.*

*Rated 1 (Strongly Disagree) to 5 (Strongly Agree).*

2. I think that I would like to use this system frequently.
3. I found the system unnecessarily complex. *(reverse scored)*
4. I thought the system was easy to use.
5. I think that I would need the support of a technical person to be able to use this system. *(reverse scored)*
6. I found the various functions in this system were well integrated.
7. I thought there was too much inconsistency in this system. *(reverse scored)*
8. I would imagine that most people would learn to use this system very quickly.
9. I found the system very cumbersome to use. *(reverse scored)*
10. I felt very confident using the system.
11. I needed to learn a lot of things before I could get going with this system. *(reverse scored)*

*Scoring: Items 2, 4, 6, 8, 10 (odd in SUS): score − 1. Items 3, 5, 7, 9, 11 (even in SUS): 5 − score. Sum all converted scores × 2.5. Range: 0–100. ≥68 = above average.*

---

### Technology Acceptance Model (TAM) — Items 12–19
*Source: Davis, F. D. (1989). Perceived usefulness, perceived ease of use, and user acceptance of information technology. MIS Quarterly, 13(3), 319–340. https://doi.org/10.2307/249008*

*Rated 1 (Strongly Disagree) to 7 (Strongly Agree). Items adapted to the automation context.*

**Perceived Usefulness (PU)**

12. This system would improve my ability to set up automations.
13. This system would increase my productivity in automating tasks.
14. This system would make it easier for me to automate tasks.
15. I find this system useful for my work.

**Perceived Ease of Use (EOU)**

16. Learning to work with this system was easy for me.
17. I found it easy to get the system to do what I wanted.
18. My interaction with the system was clear and understandable.
19. I found it easy to become proficient at using this system.

---

### Perceived Control — Items 20–22
*Source: Ajzen, I. (1991). The theory of planned behavior. Organizational Behavior and Human Decision Processes, 50(2), 179–211. https://doi.org/10.1016/0749-5978(91)90020-T. Items adapted to the automation context.*

*Rated 1 (Strongly Disagree) to 7 (Strongly Agree).*

20. I felt in control of what the system did at each step.
21. I understood what the system would do as its next step before it did it.
22. I felt that I could steer the automation in the direction I wanted.
